# Recovery Sprint

Status: Active
Date Started: 2026-07-05

## Purpose

The Recovery Sprint exists because the founding Validate conversation produced important product decisions, but the work was not reliably preserved in a durable repository.

The immediate goal is not to build Validate.

The immediate goal is to make sure Validate's founding work survives.

## Problem

The founding conversation created:

- The Validate philosophy
- The Evidence Model
- The Reasoning Framework
- The beginning of the Decision Engine
- The beginning of the Investment Model
- The role of Chief Skeptic
- The Recovery Sprint concept
- The decision to use GitHub as source of truth

But the only exported file was a bootstrap summary. That file was useful, but incomplete.

## Recovery Principle

Chat is disposable.

Canonical artifacts are not.

## Success Criteria

The Recovery Sprint succeeds when:

- The Validate repository exists in GitHub.
- Foundational RFCs are committed.
- The first ADR is committed.
- The Decision Log is committed.
- Scratch artifacts are clearly non-canonical.
- Future sessions can begin from the repository instead of chat history.

## Initial Canonical Artifacts

- RFC-001: Manifesto
- RFC-002: Evidence Model
- RFC-003: Reasoning Framework
- ADR-001: GitHub as Canonical Repository
- Decision Log
- Things We Believe Today

## Non-Canonical Artifacts

The earlier file `outputs/validate-decision-case-001.md` is a scratch artifact.

It should not be treated as canonical because it was created before the full ChatGPT context was reviewed.
